# CloudBridge

CloudBridge — это легковесный демон для интеграции облачного хранилища (Яндекс.Диск) напрямую в файловый менеджер вашего Linux (например, Thunar в Kali Linux). Он позволяет видеть все файлы из облака как обычные локальные файлы, не скачивая их целиком, пока они вам не понадобятся.

Вместо реальных файлов на диске создаются файлы-заглушки нулевого размера. При попытке работы с ними через контекстное меню CloudBridge автоматически скачивает файл, открывает его в нужной программе, а после закрытия программы загружает изменения обратно в облако.

## Основные возможности

- Гибридная синхронизация: экономит место на локальном диске, создавая только заглушки файлов нулевого размера.
- Интеграция с Thunar: специальные пункты в контекстном меню (правая кнопка мыши) для работы с облачными файлами.
- Мгновенное открытие: скачивание файла во временную папку, открытие нужным приложением и автосинхронизация изменений при закрытии.
- Офлайн-режим: возможность принудительно сохранить облачный файл на компьютере (Store Locally) для доступности без интернета и возврата обратно в облако (Restore to Cloud).
- Фоновая работа: демон самостоятельно следит за изменениями локальной и облачной папок и синхронизирует структуру.

## Установка

Для быстрой установки и настройки в Kali Linux используйте скрипт автоматической установки.

1. Откройте терминал в папке с репозиторием (например, `/media/sf_hackaton`).
2. Запустите скрипт:

```bash
chmod +x setup_kali.sh
./setup_kali.sh
```

Скрипт выполнит следующие действия:
- Установит необходимые системные зависимости (например, FUSE 3).
- Создаст виртуальное окружение Python и установит библиотеки.
- Запросит токен от Яндекс.Диска.
- Спросит, какую папку в облаке нужно синхронизировать (по умолчанию `/CloudBridgeTest`).
- Уточнит локальную папку для отображения файлов (например, `/home/kali/Videos/copypapka`).
- Настроит пункты меню в Thunar.
- Добавит глобальные команды для запуска.

Примечание: Если после установки система пишет, что команда `cloudbridge-start` не найдена, выполните команду `export PATH="$HOME/.local/bin:$PATH"` и добавьте её в свой `~/.bashrc`.

## Как запустить программу

После завершения установки вам нужно запустить фоновый демон синхронизации. В терминале выполните:

```bash
cloudbridge-start
```

Терминал с запущенным демоном нужно оставить открытым. Он будет выводить логи работы и следить за файлами. Лог также дублируется в `/tmp/cloudbridge-daemon.log`.

После запуска демона откройте вашу локальную папку (ту, которую указали во время установки) в файловом менеджере Thunar.

## Использование через контекстное меню Thunar

CloudBridge добавляет новые возможности, которые видны при клике правой кнопкой мыши на файл.

**1. Открытие облачного файла**
Файлы-заглушки весят 0 байт. Для работы с ними нажмите правой кнопкой мыши по такому файлу и выберите "Open with CloudBridge" (для новых загруженных файлов может сработать и обычный двойной клик).
- Файл скачается во временную директорию.
- Откроется программа по умолчанию для этого типа файла.
- После того, как вы закроете программу, измененный файл автоматически загрузится обратно на Яндекс.Диск.
- Если загрузка не удалась, временная копия не будет удалена, чтобы вы не потеряли данные.

**2. Сохранение файла на компьютер (Store Locally)**
Если вам нужно, чтобы файл всегда был доступен на компьютере (имел реальный размер и не зависел от интернета), нажмите правой кнопкой по заглушке и выберите "Store Locally".
- Настоящий файл будет скачан и заменит 0-байтную заглушку.
- CloudBridge перестанет выгружать изменения этого файла в облако и превращать его обратно в заглушку.

**3. Возврат к облачному режиму (Restore to Cloud)**
Если вы скачали файл локально через "Store Locally", вы можете вернуть его в режим облачной заглушки. Нажмите на него правой кнопкой и выберите "Restore to Cloud".
- Текущая локальная версия будет загружена на Яндекс.Диск.
- Локальный файл снова превратится в заглушку нулевого размера, освободив место на компьютере.

**4. Копирование публичной ссылки (Copy Link)**
Позволяет быстро сгенерировать ссылку, чтобы поделиться файлом. Нажмите правой кнопкой на заглушку или локальный файл и выберите пункт, отвечающий за копирование ссылки.
- Публичная ссылка на скачивание файла из Яндекс.Диск будет скопирована в буфер обмена.

## Ручное управление

Если вам нужно выполнить действия вручную через терминал в обход меню Thunar:

Сначала загрузите переменные окружения:
```bash
source ~/.config/cloudbridge/env
cd /media/sf_hackaton # путь к вашей директории проекта
```

**Открытие файла (аналог Open с помощью CloudBridge):**
```bash
cloudbridge-open "/путь/к/локальному/файлу"
# ИЛИ
"$CLOUDBRIDGE_PYTHON" -m src.cloud_open "/путь/к/локальному/файлу"
```

**Сохранение локально (аналог Store Locally):**
```bash
"$CLOUDBRIDGE_PYTHON" -m src.keep_local "/путь/к/локальному/файлу"
```

**Возврат в облако (аналог Restore to Cloud):**
```bash
"$CLOUDBRIDGE_PYTHON" -m src.restore_cloud "/путь/к/локальному/файлу"
```

**Скопировать публичную ссылку (аналог Copy Link):**
```bash
"$CLOUDBRIDGE_PYTHON" -m src.share_link "/путь/к/локальному/файлу"
```

Для переустановки пунктов меню Thunar:
```bash
"$CLOUDBRIDGE_PYTHON" scratch/install_thunar_action.py \
  --local-path "$LOCAL_PATH" \
  --remote-root "$YANDEX_PATH" \
  --env-file ~/.config/cloudbridge/env \
  --python-bin "$CLOUDBRIDGE_PYTHON"
thunar -q
```

## Графический интерфейс (CloudBridge Studio)

Для удобного управления настройками программы мы реализовали графический интерфейс — CloudBridge Studio.

Его можно запустить командой (если она была создана при установке):
```bash
cloudbridge-gui
```

Или вручную из папки проекта:
```bash
source ~/.config/cloudbridge/env
cd /media/sf_hackaton
"$CLOUDBRIDGE_PYTHON" -m src.gui_app
```

В графическом приложении можно:
- Изменить токен и папки синхронизации
- Настроить поведение синхронизации (период проверки, уведомления)
- Развернуть и переустановить интеграцию с Thunar
- Управлять фоновым демоном (позволяет запустить и остановить его)
- Просматривать лог работы демона в реальном времени
- Управлять списком файлов, исключённых из синхронизации (игнор-лист)
- Свернуть программу в системный трей для работы в фоне

## Требования для работы

- Linux (Kali Linux, Ubuntu)
- FUSE 3 (fuse3, libfuse3-dev)
- Python 3.9 и выше
- Файловый менеджер Thunar (для визуальной интеграции)
- Аккаунт Яндекс.Диск и выданный OAuth токен
