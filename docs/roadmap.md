# ADR-004: Интеграция с Яндекс.Диском через REST API

*   **Статус**: Принято
*   **Дата**: 2026-04-17
*   **Контекст**: Выбор приоритетного облачного провайдера для реализации нативного клиента.

## Обоснование
1.  **Популярность**: Яндекс.Диск является одним из самых популярных облачных сервисов в СНГ.
2.  **API**: Наличие развитого и документированного REST API с поддержкой OAuth2.
3.  **Производительность**: Хорошая связность серверов с локальными сетями, что обеспечивает высокую скорость загрузки.

# Дорожная карта (Roadmap) проекта

### Этап 1: Стабилизация ядра (Текущий статус)
- [ ] Базовый CLI и HybridManager.
- [ ] Реальная интеграция с Yandex REST API.
- [ ] Фоновое отслеживание файлов (Watcher).

### Этап 2: Полноценный UI и Асинхронное ядро
- [ ] Переход на **`pyfuse3`** для неблокирующей работы ФС.
- [ ] Окно авторизации (OAuth Browser flow).
- [ ] Настройка папок синхронизации через графический интерфейс (GTK4/Libadwaita).
- [ ] Виджет в трее со статусом (индикатор скорости и прогресс-бар).

### Этап 3: Глубокая системная интеграция и Кросс-платформенность
- [x] Полноценная поддержка эмблем в Nautilus.
- [x] Интеграция с **Caja** (`caja-python`) для поддержки среды MATE.
- [x] Система уведомлений о конфликтах и завершении загрузок.

### Этап 4: Индустриальная готовность и Оптимизация
- [ ] Тестирование и адаптация под **ALT Linux** (Рабочая станция).
- [ ] Сквозное шифрование (Zero-Knowledge).
- [ ] Интеллектуальный кэш (автоудаление давно не используемых локальных копий).
- [ ] Поддержка многопоточной загрузки больших файлов через `aiohttp`.
