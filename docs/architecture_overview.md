# Обзор архитектуры CloudBridge

## Концепция
CloudBridge — это гибридная система управления файлами для Linux, которая объединяет локальное хранилище и облака (Яндекс.Диск, NextCloud) в единый интерфейс. Система построена по принципу "Cloud-First, Local-Mirror", где структура папок всегда синхронизирована, но содержимое файлов загружается по запросу, с возможностью явного кеширования для офлайн-доступа.

## Основные компоненты

### 1. Core (Ядро)
- **Providers (YandexDiskProvider, NextCloudProvider)**: Изолированный слой для работы с внешними API (REST, WebDAV). Генерирует унифицированные события и списки файлов.
- **HybridManager**: Мозг системы. Отвечает за логическое объединение (Union) локального дерева папок и облачного. Решает конфликты имен и путей, управляет LRU-кешем и офлайн-файлами.
- **StateDB (SQLite)**: "Источник истины" (Single Source of Truth). Хранит текущий статус каждого файла (синхронизирован, в очереди, ошибка, доступен офлайн).

### 2. Watcher (Фоновый демон)
- Использует подсистему ядра Linux `inotify` (через библиотеку `watchdog`).
- Отслеживает изменения в реальном времени и ставит задачи на синхронизацию в очередь.

### 3. Integration Layer (Desktop & File Managers)
- Расширения на Python для проводников (GNOME Nautilus, MATE Caja).
- Обеспечивает визуальную связь с пользователем через:
    - **Emblems**: Иконки статуса поверх файлов.
    - **Context Menus**: Действия через правую кнопку мыши (генерация публичных ссылок, управление офлайн-доступом).
- **Tray & Notifications**: Системный трей и нативные уведомления о статусе синхронизации.
- **GVfs Integration**: Интеграция в боковую панель проводников (Other Locations).

## Потоки данных (Data Flows)

1.  **Событие пользователя (Drag-and-Drop)**:
    - Файл попадает в папку -> `Watcher` ловит событие -> `StateDB` помечает файл как `syncing` -> `Provider` начинает загрузку -> `StateDB` обновляет статус на `synced`.
    - `Nautilus/Caja` видит обновление в БД и меняет иконку с 🔄 на ✅.

2.  **Генерация ссылки (Share)**:
    - Пользователь нажимает ПКМ -> "Поделиться" -> Плагин запрашивает ссылку у демона -> Демон вызывает API `Provider` -> Ссылка копируется в буфер обмена и показывается уведомление.

3.  **Запуск системы (Discovery)**:
    - `HybridManager` запрашивает дерево у облака -> Сравнивает с БД -> Обновляет локальные "пустышки" (placeholder files) и синхронизирует офлайн-кеш.
