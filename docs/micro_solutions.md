# Техническое описание микро-решений

## 1. Асинхронное ядро ФС (pyfuse3)
Ядром приложения является класс `CloudBridgeFS`, наследующий `pyfuse3.Operations`. 
- **Ленивое чтение (readdir)**: При открытии папки `readdir` делает асинхронный запрос к `HybridManager`. Если данных в БД нет, инициируется `aiohttp` запрос к API Яндекса. Пользователь видит структуру папок мгновенно за счет кэширования в SQLite.
- **On-demand чтение (read)**: Системный вызов `read` перехватывается. Если файл не загружен локально, `pyfuse3` приостанавливает выполнение syscall, `aiohttp` скачивает нужный блок данных (chunk), сохраняет его в кэш и возвращает данные системе.

## 2. Асинхронное состояние (aiosqlite)
Взаимодействие с БД вынесено в отдельный слой:
- **Non-blocking Queries**: Все операции `INSERT/UPDATE` статусов файлов происходят асинхронно. Это предотвращает "заикание" файловой системы при массовом копировании файлов (например, распаковке архива).

## 3. Модульная интеграция (Caja/Nautilus)
Решение построено на принципе "Injectable Emblems":
- Расширения опрашивают локальную БД через сокет или напрямую (в режиме read-only).
- **Caja-specific**: В ALT Linux расширение использует `caja_file_info_add_emblem` для отрисовки значков синхронизации.
- **Context Actions**: При клике ПКМ расширение отправляет сигнал основному демону (например, "Force Upload"), который уже выполняет тяжелую работу.

## 4. Оптимизация сетевого трафика
- **Smart Chunking**: Использование заголовка `Range` в HTTP-запросах позволяет скачивать только те части файла, которые запрашивает ОС (например, начало файла для генерации превью).
