# ADR-001: Выбор языка программирования (Python)

*   **Статус**: Принято
*   **Дата**: 2026-04-17
*   **Контекст**: Необходимо выбрать язык для разработки системного демона и интеграции с графической оболочкой Linux.

## Рассматриваемые варианты
1.  **C/C++**: Максимальная производительность, родной язык для GNOME/GTK.
2.  **Rust**: Безопасность памяти, высокая скорость, отличная работа с FUSE.
3.  **Python**: Высокая скорость разработки, наличие `nautilus-python` для быстрой интеграции с проводником.

## Решение
Выбран **Python**.

## Обоснование
1.  **Интеграция с Nautilus**: Официальный и самый простой способ писать расширения для проводника GNOME — это `nautilus-python`. Использование другого языка потребовало бы сложного взаимодействия между процессом на C/Rust и надстройкой на Python.
2.  **Экосистема**: Наличие зрелых библиотек для сетевых запросов (`requests`) и отслеживания файловых событий (`watchdog`).
3.  **Производительность**: Для задач координации сетевых запросов и управления БД SQLite производительности Python более чем достаточно. Узкие места (криптография или тяжелый I/O) могут быть вынесены в C-расширения позже.

## Последствия
- Положительные: Очень высокая скорость прототипирования и внесения изменений.
- Отрицательные: Необходимость установки интерпретатора Python и зависимостей на целевой системе (традиционно не проблема для Linux).
