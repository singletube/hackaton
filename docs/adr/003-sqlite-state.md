# ADR-003: SQLite как хранилище состояний

*   **Статус**: Принято
*   **Дата**: 2026-04-17
*   **Контекст**: Необходимо быстро обмениваться данными о статусе файлов между фоновым демоном (Watcher) и расширением интерфейса (Nautilus).

## Рассматриваемые варианты
1.  **JSON/YAML файлы**: Легко читать, но медленно при большом количестве файлов. Проблемы с одновременным доступом.
2.  **In-memory (Redis)**: Очень быстро, но требует установки тяжелого сервиса.
3.  **SQLite**: Локальный файл, поддержка SQL, высокая скорость, встроен в стандартную библиотеку Python.

## Решение
Выбран **SQLite**.

## Обоснование
1.  **Zero Configuration**: Не требует установки сервера. Это просто файл в папке приложения.
2.  **Concurrency**: Хорошо справляется с чтением из нескольких процессов (демон пишет, Nautilus читает).
3.  **Поиск**: Позволяет мгновенно найти статус файла по его пути, что критично для отзывчивости интерфейса ("very fast response").

## Последствия
- Положительные: Надежное хранение, ACID-транзакции, легко дебажить стандартными инструментами.
- Отрицательные: Небольшой overhead на дисковое пространство (незначительно).
