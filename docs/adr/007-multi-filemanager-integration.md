# ADR-007: Мульти-платформенная интеграция (Nautilus & Caja)

*   **Статус**: Принято
*   **Дата**: 2026-04-17
*   **Контекст**: Для обеспечения "органичного" вида в различных дистрибутивах Linux (включая ALT Linux) необходимо поддерживать разные файловые менеджеры.

## Решение
Реализовать модульную систему плагинов. Основной демон CloudBridge предоставляет API через D-Bus или локальный сокет, а клиентские расширения (`nautilus-python`, `caja-python`) выступают в роли тонких визуальных оболочек.

## Обоснование
1.  **Caja (MATE)**: Критически важен для корпоративных дистрибутивов типа **ALT Linux**. Использование `caja-python` позволяет добавить кастомные пункты меню и эмблемы, идентичные Nautilus.
2.  **Архитектурная чистота**: Разделение логики (Демон на `pyfuse3`) и отображения (Плагины для FM) позволяет обновлять движок без перезагрузки графической оболочки.

## Последствия
- Положительные: Поддержка широкого спектра Linux систем.
- Отрицательные: Необходимость тестирования и поддержки нескольких кодовых баз для UI-расширений.
