# ADR-012: Интеграция с GVfs

*   **Статус**: Принято
*   **Дата**: 2026-04-17
*   **Контекст**: В дополнительных требованиях указана интеграция с GVfs, несмотря на использование FUSE (`pyfuse3`) как основного движка для гибридного вида.

## Решение
Реализовать интеграцию с GVfs на уровне закладок (gtk-bookmarks) и D-Bus, чтобы смонтированная через FUSE директория отображалась как нативный сетевой/облачный диск в боковой панели Nautilus, Thunar и других файловых менеджеров на базе GTK/GIO. Как альтернатива - предоставить кастомный GVfs-бэкенд для тех систем, где FUSE недоступен.

## Обоснование
1. **Нативная интеграция в DE**: Файловые менеджеры тесно интегрированы с GVfs. Добавление в боковую панель позволяет хранилищу выглядеть как полноценный диск в интерфейсе "Другие места" (Other Locations).
2. **Совместимость**: Пользователи привыкли, что облачные диски появляются в левой панели проводника автоматически.
3. **Гибкость**: Использование FUSE обеспечивает производительность, а GVfs-интеграция — правильное отображение в UI.

## Последствия
- Положительные: Полное соответствие требованию "Интеграция в уже существующие программы DE" и "GVfs".
- Отрицательные: Необходимость работы с GIO/GVfs API через Python (использование `PyGObject`).
