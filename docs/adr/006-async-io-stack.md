# ADR-006: Асинхронный стек I/O (aiohttp, aiosqlite)

*   **Статус**: Принято
*   **Дата**: 2026-04-17
*   **Контекст**: Для обеспечения максимальной производительности асинхронной ФС (`pyfuse3`) все внешние операции ввода-вывода также должны быть неблокирующими.

## Решение
Использование связки **`aiohttp`** для сетевых запросов и **`aiosqlite`** для работы с базой данных.

## Обоснование
1.  **aiohttp**:
    - Поддерживает `keep-alive` соединения, что критично для работы с Yandex Disk REST API.
    - Позволяет эффективно реализовывать `Range`-запросы (скачивание файлов кусками) в параллельных потоках внутри одного процесса.
2.  **aiosqlite**:
    - Переносит выполнение SQL-запросов в отдельные потоки, не блокируя основной цикл событий `asyncio`. Это гарантирует, что даже долгий сложный запрос к БД не вызовет задержку в ответе файловой системы пользователю.

## Последствия
- Положительные: Линейная масштабируемость при увеличении количества одновременных операций.
- Отрицательные: Необходимость контроля конкурентного доступа к ресурсам (mutex, semaphores).
