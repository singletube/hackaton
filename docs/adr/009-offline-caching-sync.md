# ADR-009: Кеширование, офлайн-доступ и синхронизация

*   **Статус**: Принято
*   **Дата**: 2026-04-17
*   **Контекст**: Пользователям необходима возможность работать с файлами без интернета и синхронизировать локальные директории.

## Решение
Реализовать локальный кеш файлов на базе LRU (Least Recently Used) для гибридного режима, а также механизм явного закрепления файлов/папок для офлайн-доступа (Pinning). Для синхронизации использовать двунаправленный механизм отслеживания изменений (через `watchdog` локально и polling/webhooks для облака).

## Обоснование
1. **LRU-кеш**: Позволяет переиспользовать недавно скачанные файлы, экономя трафик и ускоряя доступ.
2. **Pinning**: Решает задачу "Кеширование файлов для доступа без интернета" - пользователь может явно указать, что папка или файл должны быть доступны всегда.
3. **Двунаправленная синхронизация**: Решает задачу "Синхронизация с локальными директориями".

## Последствия
- Положительные: Повышение производительности, доступность файлов в офлайне, полноценная замена классическим клиентам.
- Отрицательные: Усложнение логики работы с файлами (разрешение конфликтов при одновременном редактировании офлайн и в облаке).
