# ADR-002: Гибридный вид (Hybrid View) вместо полной синхронизации

*   **Статус**: Принято
*   **Дата**: 2026-04-17
*   **Контекст**: Нужно решить, как отображать файлы облака на локальном диске пользователя.

## Рассматриваемые варианты
1.  **Полная синхронизация (как Dropbox Classic)**: Все файлы скачиваются на диск. Занимает много места.
2.  **Только облако (как сетевой диск)**: Доступ только при наличии интернета. Медленно для проводника.
3.  **Гибридный вид (Selective Sync / Placeholder)**: Видна вся структура папок, но файлы скачиваются только при обращении к ним.

## Решение
Выбран **Гибридный вид**.

## Обоснование
1.  **Экономия места**: Пользователь может иметь терабайты данных в облаке и работать с ними на ноутбуке с диском 128 ГБ.
2.  **Органичность**: В проводнике Linux это выглядит как обычная папка, что соответствует требованию пользователя о "нативности".
3.  **Гибкость**: Мы можем объединять (Union) локальные файлы, созданные пользователем офлайн, и структуру из облака.

## Последствия
- Положительные: Мгновенный доступ к метаданным всех файлов без затрат дискового пространства.
- Отрицательные: Сложность реализации (необходимость подмены файлов "пустышками" или использование FUSE).
