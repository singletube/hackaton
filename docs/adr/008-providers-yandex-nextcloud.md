# ADR-008: Поддержка Яндекс.Диск и NextCloud

*   **Статус**: Принято
*   **Дата**: 2026-04-17
*   **Контекст**: Требуется реализовать поддержку облачных хранилищ, в первую очередь Яндекс.Диск и NextCloud, с возможностью расширения.

## Решение
Реализовать модульную архитектуру провайдеров (Provider API), где каждый облачный сервис реализует единый интерфейс. Для Яндекс.Диска использовать REST API, для NextCloud - комбинацию WebDAV и специфичных REST API (для шаринга ссылок и т.д.).

## Обоснование
1. **Яндекс.Диск**: REST API предоставляет больше возможностей и работает стабильнее, чем их WebDAV.
2. **NextCloud**: Имеет развитый WebDAV, но для дополнительных функций (шаринг) необходимо использовать их Provisioning API / OCS API.
3. **Модульность**: Позволит легко добавлять новые облака в будущем без изменения основного движка.

## Последствия
- Положительные: Легкая масштабируемость и поддержка новых хранилищ (Google Drive, Dropbox и др.).
- Отрицательные: Необходимость поддержания абстрактного слоя, который должен учитывать особенности каждого провайдера.
